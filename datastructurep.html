Data Structures in Python



Lists:

Lists are ordered collections of items, enclosed in square brackets ([]). They can store heterogeneous data types (e.g., integers, strings, objects) and allow duplicates. Lists are mutable, meaning you can add, remove, or modify elements after creation.

Example:



    my_list = [1, 2, "three", True]


Tuples:

Tuples are similar to lists but are immutable, meaning they cannot be modified after creation. They are defined using parentheses (). Tuples are often used when you want to store related pieces of data that should not be changed.

Example:



    my_tuple = (1, 2, "three", True)


Sets:

Sets are unordered collections of unique elements, enclosed in curly braces ({}) or created using the set() function. They are useful when you need to store and operate on unique values. Sets do not support indexing or slicing since they are unordered.

Example:



    my_set = {1, 2, 3, 4}


Dictionaries:



Dictionaries are key-value pairs enclosed in curly braces ({}). They are mutable and allow you to store and retrieve values based on a specific key. Each key in a dictionary must be unique, but values can be duplicated.

Example:



    my_dict = {"name": "John", "age": 25, "city": "New York"}


Strings:



While not a data structure per se, strings are commonly used to store sequences of characters. They are immutable, meaning they cannot be modified after creation. Python provides many built-in string manipulation methods.

Example:



    my_string = "Hello, World!"


These are some of the fundamental data structures in Python. Each has its own characteristics, advantages, and use cases. Python also provides additional data structures through modules and libraries, such as arrays, stacks, queues, and more. These additional data structures can be used depending on the specific requirements of your program.
